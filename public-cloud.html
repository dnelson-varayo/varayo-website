Best for

When public cloud makes sense

Public cloud provides flexibility and speed for workloads that don't require dedicated infrastructure.

Dev/Test environments
Fast provisioning for engineering teams. Spin up environments in minutes, tear them down when done. Pay only for what you use.
Burst capacity
Scale during traffic peaks without re-architecting. Handle seasonal loads or unexpected growth without long-term commitments.
Hybrid deployments
Pair with dedicated transport for lower latency and stronger security. Direct connections bypass the public internet.
Edge workloads
Place compute closer to users and data sources. Reduce latency for real-time applications and content delivery.
Platform features

API provisioning
Automate infrastructure management with RESTful APIs. Integrate with CI/CD pipelines and infrastructure-as-code tools.

Support access

Architecture

How it's built

Geographic regions
Deploy workloads in multiple locations for redundancy and performance. Choose regions based on latency requirements and data residency needs.
Network connectivity
Public internet access included. Add private connectivity via wavelength, MPLS, or dedicated internet for hybrid architectures.
Storage options
Block storage for databases, object storage for unstructured data, and snapshots for backup. Performance tiers match workload requirements.
Security controls
Network isolation, firewalls, and encryption at rest. Security groups and access controls managed through portal or API.
